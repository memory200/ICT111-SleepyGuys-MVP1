# MVP Feature Prioritization (MoSCoW Framework)

##  Must Have (Core MVP Scope for Semester Prototype)
1. *Public Feed & Listing Cards:* Display lost/found items with title, image, location, and date.
2. *Item Search & Filtering:* Filter by Category (IDs, Keys, Tech) and Location (Library, Canteen, Block A).
3. *Report Form (Upload UI):* Input form for submitting found items with image attachment.
4. *Item Status Badge:* Clear visual indicator (Active / Claim Pending / Claimed).

---

##  Should Have (Secondary Features if Time Permits)
1. *Basic Admin/Security Panel:* Dashboard for security personnel to mark items as claimed.
2. *Simple Claim Request Button:* Prompt allowing users to submit ownership verification notes.

---

##  Could Have (Future Enhancements)
1. *Notification System:* Email notification when an item matching a keyword is posted.
2. *Interactive Campus Map:* Visual map showing exact pins where items were found.

---

##  Won't Have (Out of Scope for Lab MVP)
1. Hardware IoT Tracking / Bluetooth Tag Integration.
2. AI-based facial or biometric identification for claims.
