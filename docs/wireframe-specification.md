# Wireframe UI/UX Specification

## 1. Wireframe Inventory & Layout Specifications

### Screen 1: Homepage (`/wireframes/homepage.png`)
* **Purpose:** Introduce portal purpose, show core metrics, and provide direct CTAs.
* **UI Layout:**
  * Top Nav: Logo, Search Bar, "Report Found Item" Button.
  * Hero Section: Primary Headline, CTA Buttons ("Browse Items", "Report Found").
  * Recent Listings Preview: Grid cards showing latest found items.

---

### Screen 2: Input / Submission Form (`/wireframes/input-form.png`)
* **Purpose:** Allow users to submit found item reports with photos.
* **UI Layout:**
  * Form Fields: Item Title, Category Dropdown, Location Dropdown, Date Found, Description.
  * Drag-and-Drop Image Attachment Box.
  * Action Buttons: `Submit Listing`, `Cancel`.

---

### Screen 3: Record List View (`/wireframes/record-list.png`)
* **Purpose:** Provide keyword search and multi-criteria filters.
* **UI Layout:**
  * Left Sidebar: Filter controls (Category, Campus Building, Status).
  * Main Section: Listing Cards Grid with photo thumbnails, title, location tag, date, and status badge.

---

### Screen 4: Detail View (`/wireframes/detail-view.png`)
* **Purpose:** Display detailed record information and retrieval instructions.
* **UI Layout:**
  * High-res Image Preview.
  * Metadata Table: Title, Reference ID, Category, Found Location, Date/Time.
  * Claim Box: Security Desk location details, office hours, required ID verification steps.

---

### Screen 5: Admin View (`/wireframes/admin-view.png`)
* **Purpose:** Allow security staff to manage and update item claim states.
* **UI Layout:**
  * Data Table: Item ID, Title, Reporter Info, Current Status (`Active / Claim Pending / Claimed`).
  * Action Column: Dropdown to change status, Edit, or Mark as Claimed.

---

### Screen 6: Dashboard View (`/wireframes/dashboard.png`)
* **Purpose:** Display high-level system activity summary and statistics.
* **UI Layout:**
  * Key Metric Cards: Total Reported, Total Claimed, Pending Verification.
  * Bar/Pie Charts: Items Found by Location, Top Categories (IDs vs. Tech vs. Keys).
