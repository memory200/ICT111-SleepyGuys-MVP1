# System Requirements Specification

## 1. Functional Requirements (FR)

### FR-1: User Reporting & Item Listing
* *FR-1.1:* The system shall allow users to submit a report for a found item including title, category, location found, date, and description.
* *FR-1.2:* The system shall support image uploads for found items.
* *FR-1.3:* The system shall support reporting a lost item search inquiry.

### FR-2: Search and Filtering
* *FR-2.1:* The system shall allow users to search listings using keywords.
* *FR-2.2:* The system shall allow users to filter listings by item category (Electronics, IDs/Cards, Keys, Personal Belongings).
* *FR-2.3:* The system shall allow users to filter listings by campus building/location.

### FR-3: Claim & Verification Workflow
* *FR-3.1:* The system shall allow users to flag an item as "Claim Requested".
* *FR-3.2:* The system shall provide contact/location information for where the physical item can be claimed (e.g., Main Security Office).
* *FR-3.3:* The system shall allow authorized staff/admin to update the status of an item to "Claimed" or "Resolved".

---

## 2. Non-Functional Requirements (NFR)

### NFR-1: Usability & Accessibility
* *NFR-1.1:* The web portal shall be fully responsive across mobile, tablet, and desktop devices.
* *NFR-1.2:* The interface shall allow a user to complete an item submission within 3 simple steps.

### NFR-2: Performance & Feasibility
* *NFR-2.1:* Search and filter results shall load within 2 seconds under standard campus Wi-Fi connection.
* *NFR-2.2:* The backend database shall support basic image storage and retrieval without external hardware dependencies.
