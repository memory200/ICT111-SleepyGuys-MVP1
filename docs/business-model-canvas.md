# Business Model Canvas (BMC) - SleepyGuys Project

## Repository
- **Project URL:** https://github.com/memory200/ICT111-SleepyGuys-MVP1

## 1. Customer Segments
- **Primary Users:** Students and campus community members who lost or found personal belongings.
- **Secondary Users:** Campus security officers, facilities staff, and student council administrators.

## 2. Value Propositions
- **For Students:** A centralized, fast digital platform to report, search, track, and claim lost items effortlessly.
- **For Campus Staff:** Efficient record management, reduced physical storage clutter, and automated status tracking.

## 3. Channels
- Campus web portal link and QR code posters on bulletin boards.
- Student Telegram/Viber community groups and official campus social channels.

## 4. Customer Relationships
- Self-service reporting and status tracking.
- Admin verification and notification alerts upon claim approval.

## 5. Revenue / Sustainability Model
- Internal university-supported operational service.
- Operational efficiency savings (reduces staff manual workload and physical item storage costs).

## 6. Key Resources
- Web application interface and cloud data storage (Google Sheets / Supabase / JSON database).
- Admin role for item verification and campus notice boards.

## 7. Key Activities
- Item report collection, categorization, status verification, search indexing, and record updating.

## 8. Key Partners
- Campus Security Office, Student Affairs Department, Student Council, and Lecturers.

## 9. Cost Structure
- Domain registration and cloud hosting costs (Free-tier Vercel/GitHub Pages/Firebase).
- Development, maintenance, and promotional print material costs.
